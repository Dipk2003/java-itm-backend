OTP Testing Commands for Indian Trade Mart
==========================================

## Test 1: Test Server Connection
curl -X GET "http://localhost:8080/test/hello"

## Test 2: Test OTP with Phone Number
curl -X POST "http://localhost:8080/test/send-otp" -d "phone=9876543210"

## Test 3: Test OTP with Different Phone Number
curl -X POST "http://localhost:8080/test/send-otp" -d "phone=9123456789"

## Test 4: Test Registration with Email + Phone
curl -X POST "http://localhost:8080/auth/register" ^
  -H "Content-Type: application/json" ^
  -d "{'name':'Test User','email':'test@example.com','phone':'9876543210','password':'test123','role':'ROLE_USER'}"

## Test 5: Test Login with Email
curl -X POST "http://localhost:8080/auth/login" ^
  -H "Content-Type: application/json" ^
  -d "{'emailOrPhone':'test@example.com','password':'test123'}"

## Test 6: Test Login with Phone
curl -X POST "http://localhost:8080/auth/login" ^
  -H "Content-Type: application/json" ^
  -d "{'emailOrPhone':'9876543210','password':'test123'}"

Troubleshooting Steps:
====================
1. Check your Spring Boot console/terminal for the above output
2. If you don't see SMS output, check if phone field is being passed correctly
3. Verify sms.simulation.enabled=true in application.properties
4. Check logs for any error messages
5. Make sure your Spring Boot application is running on port 8080
6. Try restarting your Spring Boot application

PowerShell Commands:
==================
# Test server connection
Invoke-RestMethod -Uri "http://localhost:8080/test/hello" -Method Get

# Test OTP generation
Invoke-RestMethod -Uri "http://localhost:8080/test/send-otp" -Method Post -Body @{phone="9876543210"}

# Test with JSON for registration
$headers = @{"Content-Type" = "application/json"}
$body = @{
    name = "Test User"
    email = "test@example.com"
    phone = "9876543210"
    password = "test123"
    role = "ROLE_USER"
} | ConvertTo-Json

Invoke-RestMethod -Uri "http://localhost:8080/auth/register" -Method Post -Body $body -Headers $headers
